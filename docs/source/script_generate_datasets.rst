
edp_generate_datasets
=====================

Executes dataset generation tasks to produce PO.DAAC/ESDIS-ready ECCO
granules in NetCDF format.

Overview
--------

``edp_generate_datasets`` is the primary production script that executes dataset
generation tasks. It reads a task list (JSON), processes each task by loading
input data, applying transformations, adding metadata, and writing the final
NetCDF granules.

For each task, the script:

1. Loads ECCO grid, mapping factors, and metadata (shared across tasks)
2. Reads input MDS files for each variable
3. For lat/lon datasets: applies interpolation from native to lat/lon grid
4. For native datasets: applies land masking
5. Merges variables into a single dataset
6. Adds ancillary data (time bounds, coordinate bounds, precision settings)
7. Adds comprehensive metadata (global, variable, coordinate attributes)
8. Writes NetCDF file to local disk or uploads to S3


Usage
-----

.. code-block:: bash

    edp_generate_datasets --tasklist TASKLIST
                          [--keygen KEYGEN] [--profile PROFILE]
                          [-l LOG_LEVEL]


Arguments
---------

``--tasklist``
    Path or S3 URI of the JSON-formatted task list file generated by
    ``edp_create_job_task_list``.

``--keygen``
    For AWS SSO environments, path to the federated login key generation
    script.

``--profile``
    AWS credential profile name for SSO environments.

``-l, --log``
    Set logging level. Choices: ``DEBUG``, ``INFO``, ``WARNING``, ``ERROR``,
    ``CRITICAL``.
    Default: ``WARNING``


Entry Point
-----------

**Module:** ``ecco_dataset_production.apps.generate_datasets``

**Function:** ``main()``

Output Files
------------

+-----------------------------------------------+-------------------------------+
| File                                          | Description                   |
+===============================================+===============================+
| ``{dataset}_{period}_{date}_{ver}_{grid}_``   | NetCDF4 dataset granule       |
| ``{res}.nc``                                  |                               |
+-----------------------------------------------+-------------------------------+

**Output Filename Components:**

- ``{dataset}``: Dataset name (e.g., ``SEA_SURFACE_HEIGHT``)
- ``{period}``: Averaging period (``mon_mean``, ``day_mean``, ``snap``)
- ``{date}``: Center time (``YYYY-MM-DDTHH:MM:SS``)
- ``{ver}``: ECCO version (e.g., ``V4r5``)
- ``{grid}``: Grid type (``latlon`` or ``native``)
- ``{res}``: Resolution label (e.g., ``0p50deg`` or ``llc0090``)

**Example Output Filenames:**

.. code-block:: text

    OCEAN_VELOCITY_snap_1992-01-01T00:00:00_V4r5_latlon_0p50deg.nc

**NetCDF File Contents:**

Each output NetCDF file contains:

- **Data Variables:** Interpolated/masked ECCO fields
- **Coordinates:** time, latitude/longitude (or tile/j/i), Z (if 3D)
- **Bounds:** time_bnds, latitude_bnds, longitude_bnds, Z_bnds
- **Global Attributes:** ~50+ metadata fields (title, summary, DOI, etc.)
- **Variable Attributes:** long_name, standard_name, units, valid_min/max

**Example NetCDF Structure (ncdump -h):**

.. code-block:: text

    dimensions:
        time = UNLIMITED ;
        latitude = 360 ;
        longitude = 720 ;
        nv = 2 ;
    variables:
        double time(time) ;
            time:long_name = "center time of averaging period" ;
            time:bounds = "time_bnds" ;
        float latitude(latitude) ;
        float longitude(longitude) ;
        float SSH(time, latitude, longitude) ;
            SSH:long_name = "Dynamic sea surface height anomaly" ;
            SSH:units = "m" ;
            SSH:valid_min = -1.5 ;
            SSH:valid_max = 1.2 ;
        float SSHIBC(time, latitude, longitude) ;
        int time_bnds(time, nv) ;
        float latitude_bnds(latitude, nv) ;
        float longitude_bnds(longitude, nv) ;

    // global attributes:
        :title = "ECCO Sea Surface Height - Monthly Mean..." ;
        :summary = "This dataset contains monthly-averaged..." ;
        :doi = "10.5067/..." ;
        ...


Examples
--------

**Basic usage with local task list:**

.. code-block:: bash

    edp_generate_datasets \
        --tasklist ./tasklists/SSH_latlon_tasks.json \
        -l INFO

**Process S3-hosted task list with SSO authentication:**

.. code-block:: bash

    edp_generate_datasets \
        --tasklist s3://my-bucket/tasklists/SSH_latlon_tasks.json \
        --keygen /usr/local/bin/aws-login-pub.darwin.amd64 \
        --profile saml-pub \
        -l DEBUG

**Process with verbose logging:**

.. code-block:: bash

    edp_generate_datasets \
        --tasklist ./tasklists/THETA_native_tasks.json \
        -l DEBUG 2>&1 | tee processing.log

Detailed Flow Description
-------------------------

1. Task List Loading
^^^^^^^^^^^^^^^^^^^^

The script begins by loading the JSON task list:

- If the path is an S3 URI, downloads to a temporary directory first
- Parses JSON into a list of task dictionaries

2. Shared Resource Initialization
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

For performance optimization, shared resources are created once from the first
task and reused for all subsequent tasks:

**ECCOGrid:**
   Loads ECCO grid NetCDF file containing XC, YC, hFacC, drF, and native grid
   geometries (XC_bnds, YC_bnds, Z_bnds).

**ECCOMappingFactors:**
   Loads pre-computed interpolation weights (sparse matrices), land masks,
   and lat/lon grid definitions.

**ECCOMetadata:**
   Loads metadata JSON files for variable attributes, coordinate metadata,
   and global attributes.

3. Granule Generation (ecco_make_granule)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

For each task, the ``ecco_make_granule()`` function orchestrates the
complete granule creation process.

**Input Data Loading (ECCOMDSDataset):**

- Local files: Copied/symlinked to temporary directory
- S3 files: Downloaded via ``aws_s3_cp()``
- MDS Reading: Uses ``ecco_v4_py.read_bin_llc()``

**Grid Transformation:**

- *Lat/Lon Output:* Sparse matrix multiplication for bin-averaging
- *Native Output:* Applies 3D land mask (hFacC)

4. Ancillary Data (set_granule_ancillary_data)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

- **Array Precision:** Converts to float32 or float64
- **Valid Range:** Calculates ``valid_min``/``valid_max``
- **Fill Values:** Replaces NaN with NetCDF standard fill value
- **Bounds:** Adds time, lat/lon, depth bounds coordinates

5. Metadata Application (set_granule_metadata)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

- Variable metadata (long_name, standard_name, units)
- Coordinate metadata (axis labels, units, bounds)
- Global metadata (title, summary, DOI, creator info)
- Encoding options (compression, data types)

Error Handling
--------------

- Shared resource creation failures cause immediate exit (``SystemExit``)
- Individual granule errors are logged but don't stop processing
- Missing input variables for a timestep generate warnings
- S3 operations retry on transient failures


Performance Notes
-----------------

- Shared resources (grid, factors, metadata) loaded once per task list
- Temporary directories auto-cleanup after granule completion
- Memory-mapped reading for large input files
- Sparse matrix operations for efficient interpolation
- Parallel variable processing within each granule
